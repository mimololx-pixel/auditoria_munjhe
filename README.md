# auditoria_mu-jhe

Proyecto de auditoría (React + Vite + Tailwind v4 + framer-motion), derivado de la metodología
de `informe_mu-jhe`.

👉 **Lee [`CLAUDE.md`](./CLAUDE.md)** para la forma de trabajo, el stack, el flujo de GitHub/Vercel
y la **regla de oro**: el contenido evaluable vive en los `.md` de `docs_muñjhe/`, no en los
componentes React.

## Inicio rápido

```bash
npm install
npm run dev
```
